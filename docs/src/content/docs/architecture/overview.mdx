## Architecture Diagram

```
┌─────────────────────────────────────────────┐
│              Application (main.cpp)         │
│         Two-mode state machine              │
│    Measurement mode ↔ Display mode          │
├─────────────────────────────────────────────┤
│              Service Layer                  │
│  ┌──────────┬──────────┬──────────┬───────┐ │
│  │ DataService │ DisplayService │ ConnectivityService │ TimeSyncService │
│  └──────────┴──────────┴──────────┴───────┘ │
├─────────────────────────────────────────────┤
│              HAL Layer                      │
│  ┌──────────┬──────────┬──────────┬────────┐│
│  │SensorMgr │DisplayMgr│StorageMgr│ WiFiMgr││
│  │ RTCMgr   │          │          │BatteryMgr│
│  │ SettingsMgr│         │          │        ││
│  └──────────┴──────────┴──────────┴────────┘│
└─────────────────────────────────────────────┘
```

`main.cpp` **never calls hardware managers directly** — all interactions go through the service layer.

## Directory Structure

```
src/main.cpp                    # Firmware entrypoint
include/
  config.h                      # Pin definitions, timing, WiFi creds
  data_structures.h             # SensorReading, enums, shared types
  logger.h                      # LOG_INFO / LOG_ERROR macros

lib/
  hardware/                     # HAL layer — direct hardware access
    sensor_manager/             # BME280 I2C
    rtc_manager/                # ESP32Time RTC
    display_manager/            # ST7789 240x240 TFT
    storage_manager/            # SD card (SPI)
    wifi_manager/               # WiFi connection + abort callback
    battery_manager/            # MAX17048 fuel gauge
    settings_manager/           # Device settings (LittleFS persistence)
  services/                     # Service layer — orchestrates hardware
    data/                       # DataService
    display/                    # DisplayService
    connectivity/               # ConnectivityService
    time_sync_service/          # TimeSyncService (BLE + WiFi)
  test_runner/                  # Custom Unity test runner
  test_fixture/                 # setUp/tearDown dispatch
  utils/                        # DateTimeUtils, I2CScanner
```

## Firmware Modes

### Measurement Mode (timer wake)

- Display OFF, WiFi OFF
- Read sensors → store to SD → sleep
- Interval configurable via Settings (1m/5m/15m/30m/1hr), default 30 min

### Display Mode (button wake via EXT1)

- Turn on display → show dashboard with sensor data + time + battery + connectivity icon
- Navigate 3-item dashboard: Log Comfort, Menu, Sleep
- Full menu (7 items): Graph Temp/Humidity/Altitude, Settings, OTA, Sync Time, Sleep
- Settings sub-menu: Sleep Interval, NTP Sync, Back
- Sync sub-menu: Mode cycling, Sync Now, Back
- EXT1 wake on Select button (GPIO3)
