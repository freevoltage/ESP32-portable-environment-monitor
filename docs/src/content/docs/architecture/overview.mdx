`main.cpp` **never calls hardware managers directly** вҖ” all interactions go through the service layer.

## Directory Structure

```
src/main.cpp                    # Firmware entrypoint
include/
  config.h                      # Pin definitions, timing, WiFi creds
  data_structures.h             # SensorReading, enums, shared types
  logger.h                      # LOG_INFO / LOG_ERROR macros

lib/
  hardware/                     # HAL layer вҖ” direct hardware access
    sensor_manager/             # BME280 I2C
    rtc_manager/                # ESP32Time RTC
    display_manager/            # ST7789 240x240 TFT
    storage_manager/            # SD card (SPI)
    wifi_manager/               # WiFi connection
    battery_manager/            # MAX17048 fuel gauge
  services/                     # Service layer вҖ” orchestrates hardware
    data/                       # DataService
    display/                    # DisplayService
    connectivity/               # ConnectivityService
    time_sync_service/          # TimeSyncService (BLE + WiFi)
  test_runner/                  # Custom Unity test runner
  test_fixture/                 # setUp/tearDown dispatch
  utils/                        # DateTimeUtils, I2CScanner
```

## Firmware Modes

### Measurement Mode (timer wake)

- Display OFF, WiFi OFF
- Read sensors вҶ’ store to SD вҶ’ sleep
- ~2-3 seconds total, wakes every 30 minutes

### Display Mode (button wake via EXT1)

- Turn on display вҶ’ time sync (BLE/WiFi per config) вҶ’ show reading
- Navigate menu вҶ’ graphs, comfort logging, sleep
- EXT1 wake on Select button (GPIO3)
