## Components

| Component | Model | Connection |
|-----------|-------|------------|
| MCU | Adafruit Feather ESP32-C6 | — |
| Sensor | BME280 | I2C (SDA/SCL) |
| Display | 1.54" 240x240 IPS ST7789 | SPI |
| Storage | SD card module | SPI (separate from display) |
| Button A | Navigate | GPIO8 (active LOW, internal pullup) |
| Button B | Select | GPIO3 (active LOW, internal pullup) |

## Pin Assignments

| Signal | GPIO | Notes |
|--------|------|-------|
| `SD_CS` | 0 | SD card chip select |
| `TFT_LIT` | 2 | Backlight (active-HIGH) |
| `SEL_BUTTON_PIN` | 3 | Select button |
| `TFT_CS` | 5 | Display chip select |
| `TFT_RST` | 6 | Display reset |
| `TFT_DC` | 7 | Display data/command |
| `NAV_BUTTON_PIN` | 8 | Navigate button |

## Display

- **Model**: 1.54" 240x240 IPS ST7789
- **No SD slot** on the display board — SD card is wired separately
- **Backlight**: Active-HIGH (`HIGH` = ON, `LOW` = OFF)
- **Rotation**: `TFT_ROTATION = 2` (portrait flipped)

## Deep Sleep GPIO Hold

During deep sleep, GPIO2 (backlight) is held LOW via `gpio_hold_en()` to prevent backlight leakage. This is guarded by `HOLD_GPIO_IN_SLEEP` in `config.h`. An alternative is a pull-down resistor from `TFT_LIT` to GND on the display board.

## SD Card

The SD card module is connected via SPI with `SD_CS = GPIO0`. Note that this is a separate SPI bus from the display — the display uses `TFT_CS = GPIO5`.
