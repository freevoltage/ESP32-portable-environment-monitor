## Overview

The ESP32-C6 supports deep sleep to minimize power consumption. The hiking station uses two wake sources:

1. **Timer wake** — measurement mode (every 30 minutes)
2. **EXT1 wake** — display mode (button press)

## Measurement Mode

Timer wake every `MEASUREMENT_INTERVAL_SEC` (1800 = 30 minutes):

```
Wake → Read sensors → Store to SD → Deep sleep
```

- Display OFF
- WiFi OFF
- Total wake time: ~2-3 seconds
- Uses `esp_sleep_enable_timer_wakeup()`

## Display Mode

Button wake via EXT1:

```
Wake → Display on → WiFi/NTP sync → Menu → Deep sleep
```

- Display ON
- WiFi ON (for NTP sync)
- Uses `esp_sleep_enable_ext1_wakeup()` with Select button (GPIO3)

**EXT1 Configuration**:
```cpp
esp_sleep_enable_ext1_wakeup(
    (1ULL << SEL_BUTTON_PIN),
    ESP_EXT1_WAKEUP_ANY_LOW
);
```

**Note**: GPIO8 and GPIO9 are NOT RTC GPIOs on ESP32-C6 — only GPIO0-7 support EXT1 wakeup.

## GPIO Hold

During deep sleep, GPIO2 (backlight) is held LOW to prevent leakage:

```cpp
gpio_hold_en(GPIO_NUM_2);  // Hold backlight OFF before sleep
```

This is guarded by `#define HOLD_GPIO_IN_SLEEP 1` in `config.h`.

**Important**: ESP32-C6 does NOT auto-release GPIO hold after waking from deep sleep. The hold must be explicitly released at boot:

```cpp
// In setup(), before any display init:
gpio_hold_dis(static_cast<gpio_num_t>(TFT_LIT));
```

Without this, `digitalWrite(TFT_LIT, HIGH)` has no effect — the backlight stays pinned LOW.

**Alternative**: Hardware pull-down resistor from `TFT_LIT` to GND on the display board.

## Power Budget

| Mode | Display | WiFi | Duration |
|------|---------|------|----------|
| Measurement | OFF | OFF | ~2-3 sec |
| Display | ON | ON | Until sleep |

The majority of time is spent in deep sleep (~30 minutes between measurements).
